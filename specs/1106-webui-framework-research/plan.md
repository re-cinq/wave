# Implementation Plan: WebUI Framework Research

**Branch**: `1106-webui-framework-research` | **Date**: 2026-04-14 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/1106-webui-framework-research/spec.md`

## Summary

Evaluate Svelte/SvelteKit, Ripple, Astro, and htmx as replacement candidates for Wave's current Go-template + vanilla-JS webui. Produce a 4×9 comparison matrix, 1–2 proof-of-concept reimplementations of the `run_detail.html` page (the most complex page: DAG, log streaming, step cards, artifacts), and a recommendation document with migration strategy. This is research-only — no backend code changes.

## Technical Context

**Language/Version**: Go 1.22+ (existing backend, unchanged), JavaScript/TypeScript (PoC frontends)  
**Primary Dependencies**: Candidate-specific (Svelte compiler, Astro CLI, htmx library, Ripple TBD)  
**Storage**: N/A (research deliverables are markdown + PoC source code)  
**Testing**: Manual verification of PoC features; `go test ./...` to confirm no regressions  
**Target Platform**: Browser (served from Go binary via go:embed)  
**Project Type**: Research — documentation + proof-of-concept prototypes  
**Performance Goals**: PoC bundle size ≤ current baseline (~283 KB total static assets)  
**Constraints**: Single binary (go:embed), no Node.js runtime, no backend API changes  
**Scale/Scope**: 4 frameworks × 9 criteria = 36 matrix cells; 1–2 PoC implementations
