## Identified Duplicates (from code investigation)

Three event-line renderers identified during planning:

1. **`internal/tui/live_output.go:162`** — `formatEventLine(evt event.Event) string`
   Live TUI output buffer. Prefix `[stepID]`. Renders all event states. Stream activity truncated at 60 chars (fixed).

2. **`internal/tui/content.go:2184`** — `formatStoredEvent(ev state.LogRecord) string`
   Persisted-event renderer for detached pipelines (SQLite source). Same prefix and verbs as `formatEventLine` but operates on `state.LogRecord` (no `TokensIn/Out`, no `ToolName/Target`).

3. **`internal/display/progress.go:648`** — `BasicProgressDisplay.EmitProgress(ev event.Event) error`
   Stderr CLI output (non-TTY). Prefix `[HH:MM:SS]`, then arrow/symbol, then `stepID`. Lowercase verbs (`completed` vs `Completed`). Stream activity truncated to terminal width.

The renderers share concept space (state→symbol/verb mapping, duration/token formatting, contract status, stream activity) but differ in framing (timestamp vs stepID prefix, casing, truncation policy).
