# Implementation Plan: Skill Store Core

**Branch**: `381-skill-store-core` | **Date**: 2026-03-13 | **Spec**: [specs/381-skill-store-core/spec.md](spec.md)
**Input**: Feature specification from `/specs/381-skill-store-core/spec.md`

## Summary

Implement the core skill store for Wave: a SKILL.md parser (Agent Skills Specification format — YAML frontmatter + markdown body), a `Store` interface with `DirectoryStore` implementation providing CRUD operations (Read, Write, List, Delete), multi-source directory resolution with precedence, typed error handling, and path traversal prevention. All new code lives in the existing `internal/skill/` package alongside the legacy `Provisioner`/`SkillConfig` system without modifying existing files.

## Technical Context

**Language/Version**: Go 1.25+
**Primary Dependencies**: `gopkg.in/yaml.v3` (already in go.mod), standard library
**Storage**: Filesystem — `<source-dir>/<skill-name>/SKILL.md` directory convention
**Testing**: `go test` with table-driven tests, `go test -race ./...`
**Target Platform**: Linux, macOS (same as Wave)
**Project Type**: Single Go binary (internal package addition)
**Performance Goals**: List 50+ skills in <100ms (SC-008)
**Constraints**: No new dependencies, no modifications to existing skill package files
**Scale/Scope**: 3 new files in `internal/skill/`, ~600-800 lines of implementation + tests

## Implementation Design

### File 1: `internal/skill/parse.go` — Parser

**Purpose**: Parse SKILL.md files (YAML frontmatter + markdown body) into `Skill` structs and serialize them back.

**Key functions**:
- `Parse(data []byte) (Skill, error)` — full parse (frontmatter + body)
- `ParseMetadata(data []byte) (Skill, error)` — frontmatter only (for listing)
- `Serialize(skill Skill) ([]byte, error)` — write back to SKILL.md format
- `ValidateName(name string) error` — validate skill name against regex + length

**Internal helpers**:
- `splitFrontmatter(data []byte) (yamlBlock []byte, body []byte, err error)` — split on `---` delimiters
- `validateFrontmatter(fm frontmatter) error` — check required/optional field constraints
- `frontmatter` struct — raw YAML mapping with `allowed-tools` as string

**Parse algorithm**:
1. Find opening `---\n` (must be at byte 0)
2. Find closing `---\n` after the opening
3. Extract YAML block between delimiters → unmarshal into `frontmatter` struct
4. Extract everything after closing delimiter → body string
5. Validate required fields (name, description)
6. Validate name regex: `^[a-z0-9]([a-z0-9-]*[a-z0-9])?$`
7. Split `allowed-tools` string into `[]string` on whitespace
8. Map `frontmatter` → `Skill` and return

### File 2: `internal/skill/store.go` — Store Interface + DirectoryStore

**Purpose**: Define the `Store` interface and implement filesystem-backed CRUD with multi-source resolution.

**Types**:
- `Store` interface — `Read`, `Write`, `List`, `Delete`
- `DirectoryStore` struct — `sources []SkillSource`
- `SkillSource` struct — `Root string, Precedence int`
- `ParseError` struct — `Field`, `Constraint`, `Value`
- `DiscoveryError` struct — `Errors []SkillError`
- `SkillError` struct — `SkillName`, `Path`, `Err`

**DirectoryStore operations**:

`NewDirectoryStore(sources ...SkillSource) *DirectoryStore`
- Sorts sources by precedence (descending)
- Stores sorted slice

`Read(name string) (Skill, error)`
1. `ValidateName(name)` — reject invalid/traversal names
2. For each source (precedence order):
   - Build path: `filepath.Join(source.Root, name, "SKILL.md")`
   - If file exists: read, `Parse()`, set `SourcePath`, discover resources, return
3. If no source has the skill: return not-found `ParseError`

`Write(skill Skill) error`
1. Validate skill (name, description)
2. `Serialize(skill)` → bytes
3. Build dir: `filepath.Join(sources[0].Root, skill.Name)`
4. `os.MkdirAll(dir, 0755)`
5. `os.WriteFile(filepath.Join(dir, "SKILL.md"), bytes, 0644)`

`List() ([]Skill, error)`
1. Initialize `seen` map and `skills` slice
2. For each source (precedence order):
   - If source root doesn't exist: skip silently
   - Scan for `*/SKILL.md` entries
   - For each: `ParseMetadata()`, skip if name already in `seen`
   - Track parse errors in `[]SkillError`
3. If any parse errors: return `(skills, &DiscoveryError{Errors: ...})`
4. Otherwise: return `(skills, nil)`

`Delete(name string) error`
1. `ValidateName(name)` — reject invalid/traversal names
2. For each source (precedence order):
   - Build path: `filepath.Join(source.Root, name)`
   - If dir exists: `os.RemoveAll(dir)`, return nil
3. If no source has the skill: return not-found error

**Resource discovery** (internal helper):
```
discoverResources(skillDir string) []string
```
Scans for files in `scripts/`, `references/`, `assets/` subdirectories. Returns relative paths.

### File 3: `internal/skill/store_test.go` — Tests

**Purpose**: Comprehensive table-driven tests covering all parser and store operations.

**Test structure** (organized by user story priority):

1. **Parser tests** (P1 — User Story 1):
   - Valid SKILL.md with all fields
   - Valid SKILL.md with only required fields
   - Missing name → ParseError
   - Missing description → ParseError
   - No frontmatter delimiters → ParseError
   - Invalid name (uppercase, special chars) → ParseError
   - Empty file → ParseError
   - Malformed YAML → error
   - Empty body (frontmatter only) → valid
   - Compatibility field exceeding 500 chars → ParseError
   - `allowed-tools` parsing (single tool, multiple tools, empty)
   - `metadata` map parsing

2. **ParseMetadata tests** (P1):
   - Returns same frontmatter as Parse
   - Body field is empty
   - Same validation as Parse

3. **Serialize tests** (SC-002 round-trip):
   - Round-trip: Parse → Serialize → Parse produces equivalent Skill
   - Validation before serialization
   - Correct YAML + body format

4. **ValidateName tests** (FR-009):
   - Valid names: "golang", "my-skill", "a", "a-b-c"
   - Invalid: uppercase, dots, underscores, path traversal (`../etc`, `foo/bar`)
   - Boundary: 64 chars (valid), 65 chars (invalid)

5. **Discovery tests** (P2 — User Story 2):
   - Directory with multiple skills
   - Mixed valid/invalid skills
   - Empty directory
   - Non-existent directory → error
   - Name/directory mismatch → validation error

6. **List tests** (P3 — User Story 3):
   - Multiple valid skills
   - Mix of valid + invalid (returns valid + DiscoveryError)
   - No sources → empty list

7. **Read tests** (P4 — User Story 4):
   - Existing skill → full Skill returned
   - Non-existent → not-found error
   - With resource files → ResourcePaths populated
   - Path traversal name → security error

8. **Write tests** (P5 — User Story 5):
   - Valid skill → directory + SKILL.md created
   - Overwrite existing → updated
   - Invalid name → validation error
   - Empty description → validation error
   - Path traversal name → security error

9. **Delete tests** (P7 — User Story 6):
   - Existing skill → directory removed
   - Non-existent → not-found error
   - Path traversal → security error

10. **Multi-source tests** (P6 — User Story 7):
    - Higher-precedence source shadows lower
    - Skill only in lower source → returned
    - List merges with dedup

11. **Integration test** (SC-001):
    - Parse all 13 existing `.claude/skills/*/SKILL.md` files
    - Verify all parse successfully
