# Implementation Plan: Polish Log Streaming UX

**Branch**: `464-log-streaming-ux` | **Date**: 2026-03-17 | **Spec**: `specs/464-log-streaming-ux/spec.md`
**Input**: Feature specification from `/specs/464-log-streaming-ux/spec.md`

## Summary

Add a rich log viewer to the Wave web UI run detail page that renders `stream_activity` SSE events as structured, per-step collapsible log output with auto-scroll, ANSI color rendering, search with match navigation, download/copy, and SSE reconnection indicators. All changes are client-side (JS/CSS/HTML templates) — no backend modifications. The existing SSE pipeline already delivers `stream_activity` events to the browser but discards them; this feature renders them.

## Implementation Components

### Component 1: Log Viewer Core (`log-viewer.js`)

**New file**: `internal/webui/static/log-viewer.js`

Responsibilities:
- `LogViewer` class: manages all log sections, search state, connection state
- `addLine(stepId, eventData)`: creates LogLine from SSE event, buffers for batch rendering
- `flushBatch()`: rAF-driven DOM insertion, respects CSS containment
- `toggleSection(stepId)`: expand/collapse with CSS transition
- `initSections(stepCards)`: bootstraps LogSection objects from existing step card DOM

Key patterns:
- All state in single `LogViewer` instance (no globals except the instance reference)
- DOM manipulation batched via `requestAnimationFrame` (100ms frame budget)
- Each log line element gets `contain: content` for layout isolation

### Component 2: ANSI-to-HTML Converter (within `log-viewer.js`)

**Function**: `ansiToHtml(rawText)`

Supports:
- SGR codes: reset (0), bold (1), italic (3), underline (4), strikethrough (9)
- Standard 16 colors: foreground (30-37, 90-97), background (40-47, 100-107)
- 256 color palette: `38;5;N` (foreground), `48;5;N` (background)
- Strips unsupported/malformed sequences cleanly

Implementation: Single-pass regex parser (`/\x1b\[([0-9;]*)m/g`) with state machine tracking current styles. Output is HTML `<span>` elements with inline `style` attributes or CSS classes.

### Component 3: Auto-Scroll (`log-viewer.js`)

**Mechanism**: `IntersectionObserver` on a sentinel `<div>` at the bottom of each log section.

- Sentinel visible → `autoScroll = true`, new lines trigger `scrollIntoView()`
- Sentinel not visible (user scrolled up) → `autoScroll = false`, show "Jump to bottom" button
- "Jump to bottom" click → `sentinel.scrollIntoView({behavior: 'smooth'})`, re-enable auto-scroll
- Manual scroll back to bottom (sentinel re-enters viewport) → auto-re-enable

### Component 4: Search Engine (`log-viewer.js`)

**UI**: Search input bar in the run detail page header area.

- Debounced input (300ms) triggers full-text search across all expanded sections
- Case-insensitive substring match on LogLine.message
- Match highlighting via `<mark>` elements wrapping matched substrings
- Next/Prev navigation: Ctrl+G / Shift+Ctrl+G (or click buttons)
- Match counter display: "N of M matches"
- New streaming lines checked against active query; match count updated dynamically
- Clear search removes all highlights, resets navigation

### Component 5: Collapsible Step Sections (HTML + CSS + JS)

**Modify**: `step_card.html` — add a `.step-log` container inside `.step-body`.

```html
<div class="step-log" id="log-{{.StepID}}" data-step-id="{{.StepID}}">
    <div class="step-log-content"></div>
    <div class="step-log-sentinel"></div>
</div>
```

**Behavior**:
- Click on `.step-header` toggles `.step-log` visibility via `.step-collapsed` CSS class
- Running/failed steps: expanded by default
- Completed/pending steps: collapsed by default
- Header shows chevron indicator (▶ collapsed, ▼ expanded)
- Zero log lines → "No output" placeholder

### Component 6: Log Line Rendering (CSS)

**Structure per line**:
```html
<div class="log-line" style="contain: content">
    <span class="log-gutter">42</span>
    <span class="log-time">14:32:01.245</span>
    <span class="log-tool">Bash</span>
    <span class="log-content"><!-- ANSI-rendered HTML --></span>
</div>
```

**CSS properties**:
- `contain: content` on each `.log-line` for layout isolation
- `.log-gutter`: fixed-width monospace, right-aligned, accommodates up to 5 digits
- `.log-time`: fixed-width monospace, `HH:MM:SS.mmm` format
- `.log-tool`: badge-style pill showing tool name
- `.log-content`: `white-space: pre-wrap; word-break: break-all` for long line wrapping
- Theme-aware colors via existing CSS custom properties

### Component 7: Download and Copy (within `log-viewer.js`)

**Buttons**: Added to each step section header.

- **Download**: Creates a Blob from `LogSection.lines.map(l => l.rawContent).join('\n')`, triggers download as `<stepId>.log`
- **Copy**: Uses `navigator.clipboard.writeText()` with fallback to `document.execCommand('copy')`
- Both use raw content (with original ANSI codes, without HTML rendering)

### Component 8: SSE Reconnection Indicator (CSS + JS)

**Modify**: `sse.js` — enhance `onerror` and `onopen` handlers.

- On `onerror`: set `ConnectionState.status = "reconnecting"`, show banner
- On `onopen` (after reconnect): set `ConnectionState.status = "connected"`, hide banner
- After 5 failed retries: show "Connection lost — Click to retry" with manual reconnect
- Banner positioned as a fixed bar above the log sections

### Component 9: SSE Integration (`sse.js` modification)

**Modify**: `handleSSEEvent()` in `sse.js` — route `stream_activity` events to LogViewer.

```js
if (type === 'stream_activity' && window.logViewer) {
    window.logViewer.addLine(data.step_id, data);
}
```

Also update `updateStepCardState()` to trigger section expand/collapse on state transitions.
