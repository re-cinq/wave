# Implementation Plan: Guided Workflow Orchestrator TUI

**Branch**: `248-guided-workflow-tui` | **Date**: 2026-03-16 | **Spec**: `specs/248-guided-workflow-tui/spec.md`
**Input**: Feature specification from `/specs/248-guided-workflow-tui/spec.md`

## Summary

Add a guided workflow mode to the Wave TUI that activates when `wave` is run with no subcommand. The flow follows a state machine: Health Phase → Proposals ↔ FleetView → Attached → FleetView. The health phase runs infrastructure checks with live progress, auto-transitions to proposals (enhanced Suggest view), where users can launch pipelines individually or in batches. The fleet view adds archive separation and sequence grouping. All changes build on existing TUI components — no new views are created, only extensions to existing models.

## Implementation Phases

### Phase A: Foundation — Guided Mode Scaffold (FR-001, FR-014)

1. **Add `GuidedState` enum to `views.go`** — 4 states: Health, Proposals, Fleet, Attached
2. **Add `guidedMode` and `guidedState` fields to `ContentModel`** in `content.go`
3. **Add `GuidedMode` field to `LaunchDependencies`** (used by `app.go`)
4. **Set `GuidedMode=true` in `cmd/wave/main.go`** root command handler
5. **Modify `NewContentModel`** — when `guidedMode` is true, start with `ViewHealth` instead of `ViewPipelines`, eagerly initialize health list model
6. **Modify `ContentModel.Init()`** — when guided, return health list Init() commands
7. **Test**: `wave` starts showing health checks; `wave run <pipeline>` unchanged

### Phase B: Health Phase Completion & Auto-Transition (FR-001, FR-002, FR-015, FR-016)

1. **Add completion tracking to `HealthListModel`** — `totalChecks`, `completedCount` fields
2. **Emit `HealthPhaseCompleteMsg`** when `completedCount == totalChecks`
3. **Handle `HealthPhaseCompleteMsg` in `ContentModel.Update()`** — transition `guidedState` to Proposals, initialize suggest list, emit `ViewChangedMsg{View: ViewSuggest}`
4. **Add health summary string to the transition** — pass summary to suggest list for header display
5. **Test**: Health checks complete → auto-transition to proposals within 500ms

### Phase C: Tab Navigation in Guided Mode (FR-012)

1. **Modify Tab handling in `ContentModel.Update()`** — when `guidedMode` is true, toggle between `ViewSuggest` ↔ `ViewPipelines` instead of cycling through 8 views
2. **Preserve Shift+Tab** — same toggle behavior (only 2 states)
3. **Allow Tab during health phase** — skip to fleet view (FR-001 acceptance #6)
4. **Update `StatusBarModel`** — show guided-mode-specific hints (Tab: proposals/fleet)
5. **Test**: Tab toggles between Proposals and Fleet; health phase Tab skips to fleet

### Phase D: Proposals View Enhancements (FR-003, FR-006, FR-007, FR-008, FR-020)

1. **Add health summary header** to `SuggestListModel.View()` — renders at top of list
2. **Add skip (`s`) keybinding** — toggle `skipped[cursor]`, dim skipped items in render
3. **Add modify (`m`) keybinding** — activate `textinput.Model` overlay prefilled with proposal input
4. **Add DAG preview** to `SuggestDetailModel.View()` — render execution order for sequence/parallel proposals using arrow notation
5. **Enhance empty state** (FR-020) — show "No pipeline recommendations" with `n` for manual launch, Tab for fleet
6. **Test**: Navigate proposals, skip/modify work, DAG preview renders for sequence types

### Phase E: Fleet View Enhancements (FR-009, FR-010, FR-019)

1. **Add archive divider** to `PipelineListModel.buildNavigableItems()` — insert `itemKindArchiveDivider` between running and finished sections
2. **Add sequence grouping** — parse `compose:` prefix from run names, group in running section with visual indicators (pending, running, completed)
3. **Render archive divider** in `PipelineListModel.View()` — horizontal line with "Archive" label
4. **Test**: Running pipelines above divider, completed below; sequences grouped

### Phase F: View Transitions & Launch Integration (FR-004, FR-005, FR-011, FR-013, FR-017, FR-018)

1. **Launch from proposals** — handle `SuggestLaunchMsg` in guided mode: launch pipeline, transition to `GuidedStateFleet`
2. **Batch launch** — handle `SuggestComposeMsg`: call `LaunchSequence()` for multi-select, transition to fleet
3. **Attach from fleet** — Enter on running pipeline transitions `guidedState` to `GuidedStateAttached` (existing mechanism)
4. **Detach from attached** — Esc returns `guidedState` to `GuidedStateFleet` (existing mechanism)
5. **Test**: Full cycle Health → Proposals → Launch → Fleet → Attach → Detach → Proposals

### Phase G: Polish & Compatibility (SC-001 through SC-009)

1. **Verify `wave run` unchanged** (SC-007) — ensure `GuidedMode` is false for `run` command
2. **Verify 80x24 rendering** (SC-009) — test all guided views at minimum terminal size
3. **Verify existing tests pass** (SC-003) — `go test -race ./...`
4. **Performance check** (SC-001, SC-002) — first frame <200ms, transition <500ms
5. **Edge cases** — zero proposals, health timeout, terminal resize, Ctrl+C handling, no GitHub token
