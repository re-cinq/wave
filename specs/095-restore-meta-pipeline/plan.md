## Implementation Tasks

### Task 1: Add `normalizeGeneratedPipeline()` (FR-011)

**File**: `internal/pipeline/meta.go`
**What**: Add function that auto-generates `output_artifacts` for steps with `json_schema` contracts when they're missing. Call it after `loader.Unmarshal()` in both `GenerateOnly()` and `Execute()`.
**Why**: FR-011 requires auto-generation. Philosopher output is non-deterministic; enforcement must be in the executor.

### Task 2: Extend `ValidateGeneratedPipeline()` with manifest-aware persona validation

**File**: `internal/pipeline/meta.go`
**What**: Add `ValidationOption` pattern with `WithManifest()` option. When manifest is provided, validate that all step personas exist in `m.Personas` and their adapters exist in `m.Adapters`.
**Why**: Edge case 5 in spec requires detecting unknown persona references before execution.
**Backward compat**: Existing calls without options continue to work unchanged.

### Task 3: Move timeout enforcement into `MetaPipelineExecutor.Execute()`

**File**: `internal/pipeline/meta.go`
**What**: Wrap context with `context.WithTimeout` using `getTimeout(m)` at the start of `Execute()`. This ensures timeout applies regardless of caller (CLI or direct).
**Why**: Constitutional principle 11 requires bounded execution. CLI-only enforcement is insufficient.

### Task 4: Emit `meta_generate_failed` event for all failure paths

**File**: `internal/pipeline/meta.go`
**What**: Add event emission in `GenerateOnly()` and `Execute()` when semantic validation fails (not just YAML parse errors). Include the error details in the event message.
**Why**: FR-009 requires structured progress events for monitoring. Current gap: semantic validation failures are silent.

### Task 5: Add meta-specific mock adapter response

**File**: `internal/pipeline/meta.go` or `internal/pipeline/meta_test.go`
**What**: Add `buildMetaMockResponse()` that returns a well-formed `--- PIPELINE --- / --- SCHEMAS ---` response with a minimal valid pipeline (navigator → implementer). Use this in mock adapter path for testing.
**Why**: US4-AS1 requires `--mock --dry-run` to produce a structurally valid pipeline. The generic mock adapter response doesn't contain the required delimiters.

### Task 6: Add tests for new validation paths

**File**: `internal/pipeline/meta_test.go`, `cmd/wave/commands/meta_test.go`
**What**:
- Test `normalizeGeneratedPipeline()` — steps with json_schema get output_artifacts added
- Test `ValidateGeneratedPipeline()` with `WithManifest()` — unknown persona detected
- Test timeout enforcement via context cancellation
- Test `meta_generate_failed` event emission on semantic validation failure
**Why**: SC-005 requires all tests pass with `-race`. New validation paths need coverage.

### Task 7: Verify full test suite passes

**File**: All
**What**: Run `go test -race ./...` and `golangci-lint run ./...` to ensure zero regressions.
**Why**: Constitutional principle 13. FR-013.
