# Implementation Plan: Third-Party Model Providers via OpenCode Adapter

## Summary

Add dynamic model provider configuration to the OpenCode adapter, replacing the hardcoded `anthropic/claude-sonnet-4-20250514` with a `provider/model` identifier format parsed from `AdapterRunConfig.Model`. Extract shared base environment construction into a package-level function used by both Claude and OpenCode adapters, closing the credential leakage gap in OpenCode's subprocess environment.

## Implementation Approach

### Phase 1: Extract Shared Environment (foundation)

1. Create `internal/adapter/environment.go` with:
   - `BuildCuratedEnvironment(cfg AdapterRunConfig) []string` — base vars + passthrough + cfg.Env
   - `ParseProviderModel(model string) ProviderModel` — parse `provider/model` format
   - `knownModelPrefixes` map for inference
2. Refactor `ClaudeAdapter.buildEnvironment` to call `BuildCuratedEnvironment` then append telemetry vars
3. Verify all existing Claude tests pass unchanged

### Phase 2: OpenCode Model Configuration (core feature)

1. Modify `OpenCodeAdapter.prepareWorkspace` to:
   - Call `ParseProviderModel(cfg.Model)` to resolve provider and model
   - Write resolved values to `.opencode/config.json` instead of hardcoded defaults
   - Handle empty `cfg.Model` → default to `anthropic/claude-sonnet-4-20250514`
2. Add tests for model parsing edge cases:
   - `openai/gpt-4o` → provider=openai, model=gpt-4o
   - `gpt-4o` (no prefix) → provider=openai (inferred), model=gpt-4o
   - `claude-sonnet-4-20250514` → provider=anthropic (inferred)
   - `custom/my-model` → provider=custom, model=my-model
   - `provider/org/model` → provider=provider, model=org/model
   - Empty string → provider=anthropic, model=claude-sonnet-4-20250514

### Phase 3: OpenCode Curated Environment (security)

1. Modify `OpenCodeAdapter.Run` to replace `os.Environ()` with `BuildCuratedEnvironment(cfg)`
2. Add tests verifying:
   - Base vars present
   - Passthrough vars included when set
   - Canary vars do NOT leak
   - Missing passthrough vars silently skipped

### Phase 4: Documentation

1. Update `docs/reference/adapters.md` OpenCode section with:
   - Model configuration examples (provider/model format)
   - Environment passthrough for API keys
   - Complete end-to-end example
2. Update `docs/concepts/adapters.md` to remove the "OpenCode inherits full host env" caveat
