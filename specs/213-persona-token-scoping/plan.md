# Implementation Plan: Persona Token Scoping

**Branch**: `213-persona-token-scoping` | **Date**: 2026-03-16 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/213-persona-token-scoping/spec.md`

## Summary

Add per-persona token scope declarations to `wave.yaml` with preflight validation that fails fast when forge API tokens lack required permissions. Implementation creates a new `internal/scope` package for scope parsing, platform-specific resolution (GitHub/GitLab/Gitea), token introspection via forge CLIs, and aggregate violation reporting. Integrates into the executor's existing preflight phase.

## Implementation Phases

### Phase A: Manifest Schema Extension (FR-001, FR-002, FR-003)

1. Add `TokenScopes []string` field to `manifest.Persona` in `internal/manifest/types.go`
2. Add `scope.Parse(scopeStr string) (TokenScope, error)` for parsing `<resource>:<permission>[@<ENV_VAR>]`
3. Add `scope.ValidateScopes(scopes []string) []error` for manifest-time validation
4. Add validation call in `manifest.validatePersonasListWithFile` to reject invalid scope syntax during manifest loading
5. Tests: parsing valid/invalid scopes, YAML round-trip, manifest validation errors

### Phase B: Scope Resolution (FR-011)

1. Implement `ScopeResolver` with static mapping tables per forge type
2. GitHub classic PAT mapping (OAuth scope names)
3. GitLab mapping (project/personal access token scopes)
4. Gitea mapping (Gitea permission names)
5. Unknown forge / Bitbucket ‚Üí return error for caller to handle as warning
6. Tests: resolution for each forge type, unknown forge handling, permission hierarchy

### Phase C: Token Introspection (FR-012)

1. Define `TokenIntrospector` interface
2. `GitHubIntrospector`: shell out to `gh api user --include`, parse `X-OAuth-Scopes` header
3. `GitLabIntrospector`: shell out to `glab api /personal_access_tokens/self`, parse `scopes` array
4. `GiteaIntrospector`: shell out to Gitea API, parse response
5. Factory function: `NewIntrospector(forgeType) TokenIntrospector`
6. Tests: mock command execution, header parsing, error handling

### Phase D: Scope Validator & Executor Integration (FR-004, FR-005, FR-006, FR-007, FR-009, FR-010)

1. Implement `ScopeValidator.ValidatePersonas` orchestrating parse ‚Üí resolve ‚Üí introspect ‚Üí compare
2. Aggregate all violations across all personas (FR-006)
3. Generate remediation hints per violation (FR-005)
4. Check `env_passthrough` includes required token vars
5. Integrate into `executor.go` after existing preflight checks
6. Emit `"preflight"` events for scope validation results
7. Tests: end-to-end validation with mock introspector, multi-persona aggregation

### Phase E: Onboarding Integration (FR-008)

1. Modify `buildManifest` in `internal/onboarding/onboarding.go` to include `token_scopes` comments
2. Add forge-specific token creation URL in comments
3. Tests: verify generated YAML includes scope recommendations
