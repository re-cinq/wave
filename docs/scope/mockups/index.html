Onboarding-as-Session — UI dummies

Static HTML, no backend. Open files locally. Iterate before any code lands.

Order suggested: 1 → 2 → 3 → 4. Each page has a "next" link. Push back on what's wrong before service-layer refactor begins.
1 / first run
Onboarding Session
Claude-Code subprocess streams progress. Asks user questions inline. Writes .agents/* live. Sets onboarded sentinel on completion.
2 / default landing
Work Board
Unified issues / PRs / scheduled tasks. Multi-forge. Pipelines hidden. "Run" button per item.
3 / dispatch
Work Item Detail
Issue body + applicable bindings. JIT pipeline pick + run. Recent runs on this item.
4 / evolution
Evolution Proposal
Diff of generated pipeline change. Eval signals that triggered it. Approve / reject gate.

Scope

  • Dark theme only (one-shot pass).
  • Fake-real data: code-crispies repo, plausible issue titles, plausible signals.
  • SVG icons inline. No emojis (project policy).
  • No JS yet. Forms are visual only.
  • Layout/CSS prototype only — final will reuse internal/webui/templates/layout.html.

Not yet mocked

  • Dashboard variant for power users (pipelines / personas / contracts admin).
  • Run detail page (existing /runs/<id> assumed reused).
  • Schedule editor.
  • Settings / forge auth.
