Add CSV column-type inference for numeric vs string fields

open opened 2h ago by mwc enhancement parser view on gitea →
Issue body

Currently every CSV column is read as string. We should infer numeric columns at read time so downstream operations don't need an explicit cast.

Acceptance:

  • If every non-empty cell in a column parses as a number, treat the column as numeric.
  • Mixed columns stay string.
  • User can override per-column via --column-type name=int.
  • Add tests covering: pure numeric, pure string, mixed, empty cells, scientific notation.

Don't over-engineer. No date parsing yet.
